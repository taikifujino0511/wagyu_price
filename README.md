# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

* リモートリポジトリ：Webサーバー上の(共有)プロジェクト管理ディレクトリ
* ローカルリポジトリ：ローカル(個人)のプロジェクト管理ディレクトリ

## プッシュとマージの違いは何でしょうか？

* プッシュ：ローカルリポジトリの変更をリモートリポジトリに反映する。
* マージ：別のブランチの変更を現在のブランチに反映する。

## コミットとプッシュの違い

* コミット：作業（変更）内容の記録を行う、マイルストーン。
* プッシュ：ローカルリポジトリの変更をリモートリポジトリに反映する。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

* どのような変更を行なったか(追加/変更/削除)、追加変更したファイル、変更内容の概要を簡潔に記載する。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

* プルリクエストでコードレビューが発生しリモートリポジトリ上でマージするか否か。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

* どちらの変更内容を取り込むのか、はたまたどちらも取り込むのか、プロジェクト内で意思疎通をとって解消しコミットを行う。
